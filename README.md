# 小红书笔记一键复制 · 浏览器扩展

一键提取小红书笔记详情页的全部内容（标题、正文、标签、作者、图片链接、视频链接、评论），以 JSON 格式复制到剪贴板。支持批量收集多篇笔记后一键合并复制。

> 支持 **Chrome** / **Edge** · Manifest V3

---

## ✨ 功能

- 📋 在笔记详情页显示红色 **「一键复制」** 按钮
- ➕ **「加入待复制」** 按钮，收集多篇笔记后批量复制
- 📄 复制为结构化 **JSON** 格式
- 🖼️ 提取图片直链（CDN URL）
- 🎬 提取视频直链
- 💬 自动滚动加载评论（懒加载兼容）
- 🏷️ 提取话题标签
- 🔗 附带笔记原始链接
- ⚙️ 可配置最大评论获取数量（扩展图标 → 设置）
- 🔄 兼容 SPA 无刷新路由跳转

## 复制格式

### 单篇笔记

```json
{
  "标题": "笔记标题",
  "正文": "笔记正文内容...",
  "作者": "用户名",
  "话题标签": ["#话题1", "#话题2"],
  "图片": ["https://sns-img-bd.xhscdn.com/xxx"],
  "视频": "https://sns-video-bd.xhscdn.com/xxx",
  "评论": [
    { "用户": "评论者", "内容": "评论内容" }
  ],
  "链接": "https://www.xiaohongshu.com/explore/xxx"
}
```

### 批量复制（多篇）

```json
[
  { "标题": "笔记A", "正文": "...", "评论": [...] },
  { "标题": "笔记B", "正文": "...", "评论": [...] }
]
```

> 字段按需出现——无标题不输出 `标题`，图文笔记不输出 `视频`，赞数仅 `__INITIAL_STATE__` 提取时有值。

## ⚙️ 配置

点击浏览器扩展栏的小红书图标，弹出设置面板：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最大评论获取数量 | 10 | 范围 1~10000，点击复制时自动滚动评论区加载 |

改完保存后即时生效，无需刷新页面。

## 🖱️ 使用方式

### 单篇复制

1. 打开小红书笔记详情页（`/explore/xxx`）
2. 点击 **「📋 一键复制」**
3. Toast 提示加载进度 → 复制完成

### 批量收集

1. 打开笔记A → 点 **「➕ 加入待复制」** → 右上角出现悬浮面板
2. 打开笔记B → 点 **「➕ 加入待复制」**
3. 打开笔记C → 点 **「➕ 加入待复制」**
4. 在悬浮面板点 **「📋 一起复制」** → 3篇笔记合并为 JSON 数组

### 悬浮面板功能

- 📋 显示待复制列表（标题、作者、评论数）
- ✕ 单条删除
- 🗑️ 清空队列
- ✕ 关闭面板

## 🛠️ 技术架构

```
┌─────────────┐     ┌──────────────────┐     ┌──────────┐
│ content.js  │────▶│ __INITIAL_STATE__│────▶│ JSON     │
│ (注入按钮)   │     │ (SSR 数据结构)    │     │ 输出     │
│             │     │                  │     │          │
│             │────▶│ DOM 降级提取      │────▶│          │
└─────────────┘     └──────────────────┘     └──────────┘
```

| 组件 | 文件 | 说明 |
|------|------|------|
| 内容脚本 | `content.js` | 注入按钮、提取数据、复制、队列管理 |
| 样式 | `styles.css` | 按钮和 Toast 样式 |
| 设置页 | `popup.html` + `popup.js` | 扩展图标弹窗，配置评论数量 |
| 配置 | `manifest.json` | Manifest V3 |

### 提取策略

1. **Layer 1 — `__INITIAL_STATE__`**：从 React SSR 注入的全局数据对象提取（结构最完整，含图片 list、视频 stream、评论 map）
2. **Layer 2 — DOM 降级**：`__INITIAL_STATE__` 不可用或数据不匹配时，从 DOM 元素提取（同步读取，永远反映当前页面）

### SPA 兼容

- 拦截 `pushState` / `replaceState` 监听路由变化
- MutationObserver 兜底，DOM 变化自动重新注入
- 注入前验证 DOM 包含当前笔记 ID，避免读到旧内容
- 视频 URL 取 performance API 最新匹配，避免串数据
- 评论区自动滚动触发懒加载
