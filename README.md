# 小红书笔记一键复制 · 浏览器扩展

一键提取小红书笔记详情页的全部内容（标题、正文、标签、作者、图片链接、视频链接、评论），以 JSON 格式复制到剪贴板或下载为文件。支持批量收集多篇笔记后一键合并复制/下载。

> 支持 **Chrome** / **Edge** · Manifest V3

---

## ✨ 功能

- 📋 **一键复制** — 复制当前笔记为 JSON 到剪贴板
- ⬇️ **一键下载** — 下载当前笔记为 JSON 文件
- ➕ **加入待提取** — 收集多篇笔记到队列
- 💾 **一起下载** — 批量下载队列中所有笔记
- 📋 **一起复制** — 批量复制队列中所有笔记
- 📄 结构化 JSON 输出（标题、正文、作者、图片、视频、评论）
- 💬 自动滚动加载评论（懒加载兼容）
- ⚙️ 可配置最大评论获取数量
- 🔄 兼容 SPA 无刷新路由跳转

## 📦 安装

### Chrome

1. 地址栏输入 `chrome://extensions/`
2. 打开右上角 **「开发者模式」**
3. 点击 **「加载已解压的扩展程序」**
4. 选择本项目文件夹
5. ✅ 完成

### Edge

1. 地址栏输入 `edge://extensions/`
2. 打开左下角 **「开发人员模式」**
3. 点击 **「加载解压缩的扩展」**
4. 选择本项目文件夹
5. ✅ 完成

## ⚙️ 配置

点击浏览器扩展栏的小红书图标，弹出设置面板：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最大评论获取数量 | 10 | 范围 1~10000，点击复制时自动滚动评论区加载 |

改完保存后即时生效，无需刷新页面。

## 🖱️ 使用方式

### 单篇操作

```
笔记详情页操作栏：
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ 📋 一键复制   │  │ ⬇️ 一键下载   │  │ ➕ 加入待提取  │
└──────────────┘  └──────────────┘  └──────────────┘
```

- **一键复制** → JSON 复制到剪贴板
- **一键下载** → 下载 `小红书_笔记标题.json`
- **加入待提取** → 加入右上角悬浮队列

### 批量操作

```
右上角悬浮面板（可收起/展开）：
┌──────────────────────────────────────────────┐
│ 📋 待提取列表（3篇）                        ▼ │
├──────────────────────────────────────────────┤
│ 1. 笔记标题A  用户A · 12条评论              ✕ │
│ 2. 笔记标题B  用户B · 5条评论               ✕ │
│ 3. 无标题     用户C · 0条评论               ✕ │
├──────────────────────────────────────────────┤
│ 📋 一起复制（3篇）  💾 一起下载（3篇）  清空  │
└──────────────────────────────────────────────┘
```

- 点 **▼** 收起面板（只显示标题栏）
- 点 **▶** 展开面板
- 每条右侧 **✕** 单独删除
- **一起复制** → 所有笔记合并为 JSON 数组复制到剪贴板
- **一起下载** → 下载 `小红书_批量_3篇_时间戳.json`
- **清空** → 清空队列

## 📄 输出格式

### 单篇

```json
{
  "标题": "笔记标题",
  "正文": "笔记正文内容...",
  "作者": "用户名",
  "话题标签": ["#话题1", "#话题2"],
  "图片": ["https://sns-img-bd.xhscdn.com/xxx"],
  "视频": "https://sns-video-bd.xhscdn.com/xxx",
  "评论": [
    { "用户": "评论者", "内容": "评论内容" }
  ],
  "链接": "https://www.xiaohongshu.com/explore/xxx"
}
```

### 批量

```json
[
  { "标题": "笔记A", "正文": "...", "评论": [...] },
  { "标题": "笔记B", "正文": "...", "评论": [...] }
]
```

> 字段按需出现——无标题不输出 `标题`，图文笔记不输出 `视频`，赞数仅 `__INITIAL_STATE__` 提取时有值。

## 🛠️ 技术架构

| 组件 | 文件 | 说明 |
|------|------|------|
| 内容脚本 | `content.js` | 注入按钮、提取数据、复制/下载、队列管理 |
| 样式 | `styles.css` | 按钮和 Toast 样式 |
| 设置页 | `popup.html` + `popup.js` | 扩展图标弹窗，配置评论数量 |
| 配置 | `manifest.json` | Manifest V3 |

### 提取策略

1. **Layer 1 — `__INITIAL_STATE__`**：React SSR 注入的全局 JSON（结构最完整）
2. **Layer 2 — DOM 降级**：state 不可用时从 DOM 元素提取

### SPA 兼容

- 拦截 `pushState` / `replaceState` 监听路由变化
- MutationObserver 兜底
- 注入前验证 DOM 包含当前笔记 ID
- 视频 URL 取 performance API 最新匹配
- 评论区自动滚动触发懒加载

## ⚠️ 已知问题

- 小红书频繁改版，部分 CSS 选择器可能失效
- 视频 blob URL 需通过 performance API 降级获取真实链接
- SPA 导航后按钮注入有约 800ms 延迟
- 评论懒加载每次滚动间隔 600ms，大量评论需要等待
